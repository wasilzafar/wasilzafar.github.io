What We're Building

By the end of this guide you will have a fully working GPT-style language model — trained from scratch, running on a free Google Colab GPU, capable of generating coherent English text in response to a prompt. No HuggingFace, no pre-trained weights, no magic APIs: every component — tokenizer, attention heads, transformer blocks, training loop, and inference engine — is written line by line in pure PyTorch.

What you'll build: A decoder-only GPT model with ~10M parameters trained on Shakespeare's complete works. It generates text by predicting the next character given a context window. After training for ~5 minutes on a free Colab T4 GPU, it produces surprisingly Shakespearean-sounding prose.

How Language Models Work

A language model assigns a probability to sequences of tokens. For generation, we use the autoregressive formulation: the probability of a sequence is the product of conditional probabilities — P(token at position t | all tokens before t). Training maximises this probability on real text; inference samples from it iteratively to grow a sequence.

Analogy: Autocomplete on Steroids

Your phone's keyboard predicts the next word using a 2-gram or 3-gram statistical model. A language model does the same thing but uses all preceding context (not just the last 2–3 words) via a transformer's attention mechanism, and predicts over a vocabulary of 50,000+ tokens instead of a few hundred. The "from scratch" version we build here uses characters as tokens — much simpler, but the same fundamental idea.

We use a decoder-only transformer (the GPT family) — no encoder, no cross-attention. The only constraint is causal masking: at each position, the model can only attend to tokens at equal or earlier positions. This prevents the model from "cheating" by looking at future tokens during training, and makes generation natural (you always have the past, never the future).

For background on the transformer architecture and the attention mechanism, see our dedicated deep-dives: NLP Part 9: Transformers & Attention and Attention Is All You Need — Transformer Explained.

Character Tokenizer

We build a character-level tokenizer: the vocabulary is every unique character that appears in the training text. No byte-pair encoding, no SentencePiece — just a dictionary mapping each character to an integer index. This keeps the code minimal and lets us focus on the model architecture. The trade-off: sequences are longer (each character is one token, not each word), so the model must learn longer-range dependencies.

Going further with tokenization: For production LLMs, character-level tokenization is impractical (too many tokens per sentence). BPE and WordPiece tokenizers pack more meaning per token. See NLP Part 2: Training Tokenizers from Scratch for a deep dive into building subword tokenizers with HuggingFace and SentencePiece. 
# Cell 3 — Build character-level tokenizer
import urllib.request, os

# Load text
DATA_PATH = 'shakespeare.txt'
with open(DATA_PATH, 'r', encoding='utf-8') as f:
    text = f.read()

# Build vocabulary: every unique character in the corpus
chars = sorted(list(set(text)))
vocab_size = len(chars)
print(f'Vocabulary size: {vocab_size} unique characters')
print(f'Characters: {"".join(chars[:30])}...' if len(chars) > 30 else f'Characters: {"".join(chars)}')

# Mappings: character ↔ integer
char_to_idx = {ch: i for i, ch in enumerate(chars)}
idx_to_char = {i: ch for i, ch in enumerate(chars)}

def encode(s):
    """Convert a string into a list of integer token IDs."""
    return [char_to_idx[c] for c in s]

def decode(ids):
    """Convert a list of integer token IDs back into a string."""
    return ''.join(idx_to_char[i] for i in ids)

# Verify round-trip

decoded = decode(encoded)
print(f'\nOriginal:  {sample}')
print(f'Encoded:   {encoded[:15]}...')
print(f'Decoded:   {decoded}')
print(f'Round-trip OK: {sample == decoded}')

Dataset and DataLoader

The language modelling task: given a sequence of tokens [t0, t1, ..., tN], predict the next token at each position. The input is [t0..tN-1] and the target is [t1..tN] (shifted by one). We slice the training corpus into overlapping windows of length context_length and return random batches.

# Cell 4 — Dataset and DataLoader

class ShakespeareDataset(Dataset):

        self.data = data
        self.context_len = context_len

        return len(self.data) - self.context_len

        # x: tokens [idx .. idx+context_len-1]
        # y: tokens [idx+1 .. idx+context_len]  (shifted by 1)
        x = self.data[idx : idx + self.context_len]
        y = self.data[idx + 1 : idx + self.context_len + 1]
        return x, y

Model Architecture

We implement a decoder-only transformer — the same architectural family as GPT-2 and GPT-3. No encoder, no cross-attention. The flow: token IDs → token embeddings + positional embeddings → N stacked transformer blocks → linear head → logits over vocabulary → cross-entropy loss.

GPT Decoder Architecture
flowchart TD
    A["Input tokens [t0, t1, ..., tN]"] --> B["Token Embedding (vocab_size → d_model)"]
    A --> C["Positional Embedding (context_len → d_model)"]
    B --> D["Add: token + positional embeddings"]
    C --> D
    D --> E["Transformer Block × N\n(LayerNorm → MHA → LayerNorm → FFN)"]
    E --> F["LayerNorm (final)"]
    F --> G["Linear Head (d_model → vocab_size)"]
    G --> H["Logits → Softmax → Next token probabilities"]
    H --> I["Cross-Entropy Loss (training)\nor Sample (inference)"]
                            
# Cell 5 — Token + Positional Embeddings
import torch
import torch.nn as nn

# Hyperparameters (used throughout)
VOCAB_SIZE   = 65    # Set after building tokenizer (run Cell 3 first)
D_MODEL      = 256   # Embedding dimension
N_HEADS      = 8     # Attention heads
N_LAYERS     = 6     # Transformer blocks
CONTEXT_LEN  = 256   # Max sequence length
DROPOUT      = 0.1   # Dropout rate
D_FF         = D_MODEL * 4  # Feed-forward hidden dim (1024)

class TokenAndPositionalEmbedding(nn.Module):
    """
    Combines token embeddings (learned) and positional embeddings (learned).
    GPT uses learned positional embeddings; original "Attention is All You Need" 
    used sinusoidal. Both work; learned embeddings are simpler to implement.
    """
    def __init__(self, vocab_size, d_model, context_len, dropout=0.1):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb   = nn.Embedding(context_len, d_model)
        self.dropout   = nn.Dropout(dropout)
    
    def forward(self, x):
        # x: (batch, seq_len)
        B, T = x.shape
        tok = self.token_emb(x)                                     # (B, T, d_model)
        positions = torch.arange(T, device=x.device)               # (T,)
        pos = self.pos_emb(positions)                               # (T, d_model) → broadcast to (B, T, d_model)
        return self.dropout(tok + pos)

Causal Self-Attention Head

The heart of the transformer. Each attention head computes queries, keys, and values from the input, then attends each position to all earlier positions. The causal mask (lower triangular) blocks future positions, enforcing the autoregressive property. For the full mathematical derivation, see NLP Part 9: Transformers & Attention.

# Cell 6 — Single Causal Self-Attention Head
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class CausalSelfAttentionHead(nn.Module):
    """
    One attention head with causal (lower-triangular) masking.
    
    Scaled dot-product attention:
        Attention(Q, K, V) = softmax(QK^T / sqrt(d_head)) * V
    
    The causal mask zeroes out attention to future positions,
    ensuring token at position i can only attend to positions 0..i.
    """
    def __init__(self, d_model, head_dim, context_len, dropout=0.1):
        super().__init__()
        self.head_dim = head_dim
        self.q_proj = nn.Linear(d_model, head_dim, bias=False)
        self.k_proj = nn.Linear(d_model, head_dim, bias=False)
        self.v_proj = nn.Linear(d_model, head_dim, bias=False)
        self.dropout = nn.Dropout(dropout)
        # Causal mask: lower triangular matrix of ones (registered as buffer so it
        # moves to GPU with .to(device) but is NOT a learnable parameter)
        self.register_buffer(
            'mask',
            torch.tril(torch.ones(context_len, context_len))
        )
    
    def forward(self, x):
        B, T, C = x.shape          # batch, sequence length, d_model
        Q = self.q_proj(x)         # (B, T, head_dim)
        K = self.k_proj(x)         # (B, T, head_dim)
        V = self.v_proj(x)         # (B, T, head_dim)
        
        # Scaled dot-product attention
        scale = math.sqrt(self.head_dim)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / scale  # (B, T, T)
        
        # Apply causal mask: set future positions to -inf before softmax
        scores = scores.masked_fill(self.mask[:T, :T] == 0, float('-inf'))
        
        # Softmax → attention weights (future positions become 0 after softmax)
        attn_weights = F.softmax(scores, dim=-1)       # (B, T, T)
        attn_weights = self.dropout(attn_weights)
        
        # Weighted sum of values
        out = torch.matmul(attn_weights, V)            # (B, T, head_dim)
        return out

# Test one attention head
head = CausalSelfAttentionHead(d_model=256, head_dim=32, context_len=256)
x_test = torch.randn(4, 32, 256)   # (batch=4, seq=32, d_model=256)
out = head(x_test)
print(f'Attention head output: {out.shape}')  # (4, 32, 32)
print(f'Head dim = d_model / n_heads = 256 / 8 = {256 // 8}')

Multi-Head Attention

Multi-head attention runs N independent attention heads in parallel, then concatenates their outputs and projects back to d_model. Each head can specialise: one head might capture subject-verb agreement, another syntax, another long-range coreference. This is the same principle GPT-2 and GPT-3 use — they just have many more heads and much larger d_model.

# Cell 7 — Multi-Head Causal Self-Attention
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class CausalSelfAttentionHead(nn.Module):
    def __init__(self, d_model, head_dim, context_len, dropout=0.1):
        super().__init__()
        self.head_dim = head_dim
        self.q_proj = nn.Linear(d_model, head_dim, bias=False)
        self.k_proj = nn.Linear(d_model, head_dim, bias=False)
        self.v_proj = nn.Linear(d_model, head_dim, bias=False)
        self.dropout = nn.Dropout(dropout)
        self.register_buffer('mask', torch.tril(torch.ones(context_len, context_len)))
    
    def forward(self, x):
        B, T, C = x.shape
        Q, K, V = self.q_proj(x), self.k_proj(x), self.v_proj(x)
        scale = math.sqrt(self.head_dim)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / scale
        scores = scores.masked_fill(self.mask[:T, :T] == 0, float('-inf'))
        attn_weights = F.softmax(scores, dim=-1)
        return torch.matmul(self.dropout(attn_weights), V)

class MultiHeadCausalSelfAttention(nn.Module):
    """
    N attention heads running in parallel.
    Each head has dimension d_model // n_heads.
    Outputs are concatenated and projected back to d_model.
    """
    def __init__(self, d_model, n_heads, context_len, dropout=0.1):
        super().__init__()
        assert d_model % n_heads == 0, "d_model must be divisible by n_heads"
        head_dim = d_model // n_heads
        # One head module per attention head
        self.heads = nn.ModuleList([
            CausalSelfAttentionHead(d_model, head_dim, context_len, dropout)
            for _ in range(n_heads)
        ])
        # Output projection: concat(heads) → d_model
        self.out_proj = nn.Linear(d_model, d_model)
        self.dropout  = nn.Dropout(dropout)
    
    def forward(self, x):
        # Run all heads, concat along last dim: (B, T, n_heads * head_dim)
        head_outputs = torch.cat([h(x) for h in self.heads], dim=-1)
        # Project back: (B, T, d_model)
        return self.dropout(self.out_proj(head_outputs))

# Test multi-head attention

x_test = torch.randn(4, 32, 256)
out = mha(x_test)
print(f'Multi-head attention output: {out.shape}')  # (4, 32, 256)

Position-Wise Feed-Forward Network

After attention, each token's representation passes through a small MLP independently. This is the "thinking" step: attention routes information between tokens; the FFN refines each token's representation based on that information. The hidden dimension is typically 4 × d_model. We use GELU activation (smoother than ReLU; used by GPT-2/3).

# Cell 8 — Position-Wise Feed-Forward Network
import torch
import torch.nn as nn

class FeedForward(nn.Module):
    """
    Two-layer MLP applied to each token position independently.
    
    Architecture: Linear(d_model → d_ff) → GELU → Dropout → Linear(d_ff → d_model)
    where d_ff = 4 * d_model (standard expansion factor)
    """
    def __init__(self, d_model, d_ff=None, dropout=0.1):
        super().__init__()
        if d_ff is None:
            d_ff = 4 * d_model
        self.net = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),          # Gaussian Error Linear Unit — smoother than ReLU
            nn.Dropout(dropout),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )
    
    def forward(self, x):
        return self.net(x)   # Applied identically to every token position

# Test
d_model, d_ff = 256, 1024
ffn = FeedForward(d_model, d_ff)
x_test = torch.randn(4, 32, d_model)

Transformer Block

One transformer block combines multi-head attention and the FFN with residual connections and Pre-LayerNorm. GPT-2 uses Pre-LN (normalise before attention/FFN) rather than the original Post-LN — it trains more stably, especially at depth. Residual connections allow gradients to flow directly through the stack, enabling training of very deep networks.

# Cell 9 — Transformer Block (Pre-LayerNorm variant)
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

# Reuse the classes defined above (paste them here if running standalone)
class CausalSelfAttentionHead(nn.Module):
    def __init__(self, d_model, head_dim, context_len, dropout=0.1):
        super().__init__()
        self.head_dim = head_dim
        self.q_proj = nn.Linear(d_model, head_dim, bias=False)
        self.k_proj = nn.Linear(d_model, head_dim, bias=False)
        self.v_proj = nn.Linear(d_model, head_dim, bias=False)
        self.dropout = nn.Dropout(dropout)
        self.register_buffer('mask', torch.tril(torch.ones(context_len, context_len)))
    def forward(self, x):
        B, T, C = x.shape
        Q, K, V = self.q_proj(x), self.k_proj(x), self.v_proj(x)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.head_dim)
        scores = scores.masked_fill(self.mask[:T, :T] == 0, float('-inf'))
        return torch.matmul(self.dropout(F.softmax(scores, dim=-1)), V)

class MultiHeadCausalSelfAttention(nn.Module):
    def __init__(self, d_model, n_heads, context_len, dropout=0.1):
        super().__init__()
        head_dim = d_model // n_heads
        self.heads = nn.ModuleList([CausalSelfAttentionHead(d_model, head_dim, context_len, dropout) for _ in range(n_heads)])
        self.out_proj = nn.Linear(d_model, d_model)
        self.dropout  = nn.Dropout(dropout)
    def forward(self, x):
        return self.dropout(self.out_proj(torch.cat([h(x) for h in self.heads], dim=-1)))

class FeedForward(nn.Module):
    def __init__(self, d_model, dropout=0.1):
        super().__init__()
        self.net = nn.Sequential(nn.Linear(d_model, 4*d_model), nn.GELU(), nn.Dropout(dropout), nn.Linear(4*d_model, d_model), nn.Dropout(dropout))
    def forward(self, x):
        return self.net(x)

class TransformerBlock(nn.Module):
    """
    Pre-LayerNorm transformer block:
        x = x + MHA(LayerNorm(x))   # Self-attention with residual
        x = x + FFN(LayerNorm(x))   # Feed-forward with residual
    """
    def __init__(self, d_model, n_heads, context_len, dropout=0.1):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model)
        self.attn  = MultiHeadCausalSelfAttention(d_model, n_heads, context_len, dropout)
        self.norm2 = nn.LayerNorm(d_model)
        self.ffn   = FeedForward(d_model, dropout)
    
    def forward(self, x):
        x = x + self.attn(self.norm1(x))   # Attention with residual
        x = x + self.ffn(self.norm2(x))    # FFN with residual
        return x

# Test transformer block
block = TransformerBlock(d_model=256, n_heads=8, context_len=256)

# Cell 10 — Full GPT Model Assembly
# (All component classes included for standalone execution)
import torch, torch.nn as nn, torch.nn.functional as F, math

class CausalSelfAttentionHead(nn.Module):
    def __init__(self, d, hd, ctx, dr=0.1):
        super().__init__(); self.hd=hd; self.qkv=[nn.Linear(d,hd,bias=False) for _ in range(3)]; [setattr(self,n,l) for n,l in zip(['q','k','v'],self.qkv)]; self.dp=nn.Dropout(dr); self.register_buffer('mask',torch.tril(torch.ones(ctx,ctx)))
    def forward(self,x):
        B,T,C=x.shape; Q,K,V=self.q(x),self.k(x),self.v(x); s=torch.matmul(Q,K.transpose(-2,-1))/math.sqrt(self.hd); s=s.masked_fill(self.mask[:T,:T]==0,float('-inf')); return torch.matmul(self.dp(F.softmax(s,dim=-1)),V)

class MultiHeadCausalSelfAttention(nn.Module):
    def __init__(self, d, nh, ctx, dr=0.1):
        super().__init__(); hd=d//nh; self.heads=nn.ModuleList([CausalSelfAttentionHead(d,hd,ctx,dr) for _ in range(nh)]); self.proj=nn.Linear(d,d); self.dp=nn.Dropout(dr)
    def forward(self,x): return self.dp(self.proj(torch.cat([h(x) for h in self.heads],dim=-1)))

class FeedForward(nn.Module):
    def __init__(self, d, dr=0.1):
        super().__init__(); self.net=nn.Sequential(nn.Linear(d,4*d),nn.GELU(),nn.Dropout(dr),nn.Linear(4*d,d),nn.Dropout(dr))
    def forward(self,x): return self.net(x)

class TransformerBlock(nn.Module):
    def __init__(self, d, nh, ctx, dr=0.1):
        super().__init__(); self.n1=nn.LayerNorm(d); self.attn=MultiHeadCausalSelfAttention(d,nh,ctx,dr); self.n2=nn.LayerNorm(d); self.ff=FeedForward(d,dr)
    def forward(self,x): return (x:=x+self.attn(self.n1(x)))+self.ff(self.n2(x))

class MiniGPT(nn.Module):
    """
    Decoder-only GPT model from scratch.
    
    Architecture:
        - Token embedding: vocab_size → d_model
        - Positional embedding: context_len → d_model
        - N × TransformerBlock
        - Final LayerNorm
        - Linear language model head: d_model → vocab_size
    """
    def __init__(self, vocab_size, d_model=256, n_heads=8, n_layers=6,
                 context_len=256, dropout=0.1):
        super().__init__()
        self.context_len = context_len
        
        # Embeddings
        self.tok_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(context_len, d_model)
        self.emb_dropout = nn.Dropout(dropout)
        
        # Transformer blocks
        self.blocks = nn.Sequential(*[
            TransformerBlock(d_model, n_heads, context_len, dropout)
            for _ in range(n_layers)
        ])
        
        # Final normalisation + LM head
        self.norm = nn.LayerNorm(d_model)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)
        
        # Weight tying: share token embedding and LM head weights
        # (Used in GPT-2; reduces parameters and improves performance)
        self.lm_head.weight = self.tok_emb.weight
        
        # Initialise weights (GPT-2 style)
        self.apply(self._init_weights)
    
    def _init_weights(self, module):
        if isinstance(module, (nn.Linear, nn.Embedding)):
            nn.init.normal_(module.weight, mean=0.0, std=0.02)
        if isinstance(module, nn.Linear) and module.bias is not None:
            nn.init.zeros_(module.bias)
        if isinstance(module, nn.LayerNorm):
            nn.init.ones_(module.weight)
            nn.init.zeros_(module.bias)
    
    def forward(self, idx, targets=None):
        """
        idx:     (B, T) — integer token indices
        targets: (B, T) — target indices for loss computation (optional)
        
        Returns: (logits, loss) where loss is None if targets not provided
        """
        B, T = idx.shape
        assert T <= self.context_len, f"Sequence length {T} > context_len {self.context_len}"
        
        positions = torch.arange(T, device=idx.device)        # (T,)
        x = self.emb_dropout(
            self.tok_emb(idx) + self.pos_emb(positions)       # (B, T, d_model)
        )
        x = self.blocks(x)                                     # (B, T, d_model)
        x = self.norm(x)                                       # (B, T, d_model)
        logits = self.lm_head(x)                               # (B, T, vocab_size)
        
        loss = None
        if targets is not None:
            # Flatten for cross-entropy: (B*T, vocab_size) vs (B*T,)
            loss = F.cross_entropy(
                logits.view(-1, logits.size(-1)),
                targets.view(-1)
            )
        return logits, loss

Training on Google Colab

The training loop iterates over batches, computes cross-entropy loss between predicted and actual next tokens, backpropagates, and updates weights with AdamW. We use gradient clipping (prevents exploding gradients common in transformers) and a cosine learning rate schedule (warms up, then decays — used by GPT-2/3).

Training time on free Colab T4 GPU: ~5–8 minutes for 5,000 iterations with the default hyperparameters. You'll see the loss drop from ~4.3 (random) to ~1.5–1.6 (coherent Shakespeare). For scale reference: GPT-3 trained for ~355 GPU-years. We're doing this in 5 minutes on a single T4. 
# Cell 11 — Training Loop (run after Cell 10 — requires model, train_loader, val_loader)
import torch, torch.nn.functional as F, time, math
import matplotlib.pyplot as plt
from torch.optim import AdamW
from torch.optim.lr_scheduler import CosineAnnealingLR

# Training hyperparameters
MAX_ITERS      = 5000     # Total training iterations
EVAL_INTERVAL  = 500      # Evaluate on val set every N iters
LR_MAX         = 3e-4     # Peak learning rate
LR_MIN         = 1e-5     # Minimum LR at end of cosine decay
WARMUP_ITERS   = 200      # Linear LR warmup
GRAD_CLIP      = 1.0      # Gradient clipping norm
BATCH_SIZE     = 64       # Mini-batch size

# Optimizer — AdamW with weight decay (regularisation)
optimizer = AdamW(model.parameters(), lr=LR_MAX, weight_decay=0.1, betas=(0.9, 0.95))
scheduler = CosineAnnealingLR(optimizer, T_max=MAX_ITERS, eta_min=LR_MIN)

def lr_warmup_schedule(optimizer, step, warmup_steps, lr_max):
    """Linear warmup: LR ramps from 0 to lr_max over warmup_steps."""
    if step < warmup_steps:
        lr = lr_max * (step + 1) / warmup_steps
        for pg in optimizer.param_groups:
            pg['lr'] = lr

@torch.no_grad()
def estimate_loss(model, loader, device, max_batches=20):
    """Estimate mean loss on a dataloader (no gradient updates)."""
    model.eval()
    total_loss, count = 0, 0
    for i, (x, y) in enumerate(loader):
        if i >= max_batches:
            break
        x, y = x.to(device), y.to(device)
        _, loss = model(x, y)
        total_loss += loss.item()
        count += 1
    model.train()
    return total_loss / count if count else float('inf')

# --- Training loop ---
train_losses, val_losses, lr_history = [], [], []
iter_num = 0
train_iter = iter(train_loader)
start_time = time.time()

model.train()
print(f'Starting training — {MAX_ITERS} iterations')
print(f'{"Iter":>6} | {"Train Loss":>11} | {"Val Loss":>9} | {"LR":>9} | {"Time/iter":>10}')
print('-' * 60)

for iter_num in range(MAX_ITERS):
    # LR warmup
    lr_warmup_schedule(optimizer, iter_num, WARMUP_ITERS, LR_MAX)
    
    # Get next batch (cycle through DataLoader)
    try:
        x, y = next(train_iter)
    except StopIteration:
        train_iter = iter(train_loader)
        x, y = next(train_iter)
    
    x, y = x.to(device), y.to(device)
    
    # Forward + backward
    optimizer.zero_grad()
    _, loss = model(x, y)
    loss.backward()
    
    # Gradient clipping: prevents exploding gradients
    torch.nn.utils.clip_grad_norm_(model.parameters(), GRAD_CLIP)
    
    optimizer.step()
    if iter_num >= WARMUP_ITERS:
        scheduler.step()
    
    train_losses.append(loss.item())
    lr_history.append(optimizer.param_groups[0]['lr'])
    
    # Periodic evaluation
    if (iter_num + 1) % EVAL_INTERVAL == 0:
        val_loss  = estimate_loss(model, val_loader, device)
        train_avg = sum(train_losses[-EVAL_INTERVAL:]) / EVAL_INTERVAL
        val_losses.append(val_loss)
        elapsed   = time.time() - start_time
        per_iter  = elapsed / (iter_num + 1) * 1000  # ms
        print(f'{iter_num+1:>6} | {train_avg:>11.4f} | {val_loss:>9.4f} | {optimizer.param_groups[0]["lr"]:>9.6f} | {per_iter:>8.1f}ms')

print(f'\nTraining complete! Total time: {(time.time()-start_time)/60:.1f} min')
print(f'Final train loss: {train_losses[-1]:.4f}')
print(f'Final val loss:   {val_losses[-1]:.4f}')

Text Generation & Inference

Generation is the autoregressive loop: feed the model a context, sample the next token from the predicted distribution, append it to the context, repeat. We implement two sampling strategies: greedy (always pick the highest-probability token — deterministic, often repetitive) and temperature + top-k sampling (sample from the top-k most likely tokens, scaled by temperature — more creative, controllable randomness).

# Cell 13 — Text generation with temperature + top-k sampling
import torch
import torch.nn.functional as F

@torch.no_grad()
def generate(model, prompt_ids, max_new_tokens=200, temperature=1.0, top_k=None, device='cpu'):
    """
    Generate text autoregressively.
    
    Args:
        model:          Trained MiniGPT
        prompt_ids:     List[int] — encoded prompt tokens
        max_new_tokens: How many new tokens to generate
        temperature:    > 1.0 = more random, < 1.0 = more focused, 0 → greedy
        top_k:          Only sample from the top-k most likely tokens (None = all)
    
    Returns:
        List[int] — full sequence (prompt + generated tokens)
    """
    model.eval()
    context = torch.tensor(prompt_ids, dtype=torch.long, device=device).unsqueeze(0)  # (1, T)
    
    for _ in range(max_new_tokens):
        # Crop context to context_len (sliding window)
        ctx = context[:, -model.context_len:]
        
        # Forward pass — only need logits for the last token
        logits, _ = model(ctx)
        logits = logits[:, -1, :]    # (1, vocab_size) — last position only
        
        if temperature == 0:
            # Greedy decoding
            next_token = logits.argmax(dim=-1, keepdim=True)
        else:
            # Temperature scaling: divide logits by T before softmax
            logits = logits / temperature
            
            # Top-k filtering: zero out logits outside top-k
            if top_k is not None:
                values, _ = torch.topk(logits, min(top_k, logits.size(-1)))
                threshold = values[:, -1].unsqueeze(-1)
                logits = logits.masked_fill(logits < threshold, float('-inf'))
            
            # Sample from the distribution
            probs = F.softmax(logits, dim=-1)
            next_token = torch.multinomial(probs, num_samples=1)  # (1, 1)
        
        context = torch.cat([context, next_token], dim=1)
    
    return context[0].tolist()

Interactive Chat Interface

Our model is a language model, not a chat model — it continues text rather than following instructions. To simulate a conversational interface, we format the prompt as a dialogue template and let the model complete it. This is exactly how early chatbots worked before RLHF and instruction tuning. For true instruction-following, you'd need to fine-tune with the methods covered in NLP Part 16: RLHF & Instruction Tuning.

# Cell 14 — Simple interactive chat interface (run in Colab)
import torch, torch.nn.functional as F

@torch.no_grad()
def generate(model, prompt_ids, max_new_tokens=200, temperature=0.8, top_k=40, device='cpu'):
    """Generate tokens autoregressively from a prompt."""
    model.eval()
    context = torch.tensor(prompt_ids, dtype=torch.long, device=device).unsqueeze(0)
    for _ in range(max_new_tokens):
        ctx = context[:, -model.context_len:]
        logits, _ = model(ctx)
        logits = logits[:, -1, :] / temperature
        if top_k:
            vals, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            logits = logits.masked_fill(logits < vals[:, -1:], float('-inf'))
        next_tok = torch.multinomial(F.softmax(logits, dim=-1), 1)
        context = torch.cat([context, next_tok], dim=1)
    return context[0].tolist()

def chat(model, encode, decode, device, temperature=0.8, top_k=40, max_response=200):
    """
    Simple text-completion chat loop.
    
    The model doesn't truly "understand" instructions — it continues the
    dialogue template based on patterns learned from Shakespeare.
    Run in Google Colab; type 'quit' to exit.
    """
    print("MiniGPT Chat (Shakespeare-style text completion)")
    print("NOTE: Model completes your prompt — it's a language model, not a chatbot.")
    print("Type 'quit' to exit.\n")
    
    history = ""  # Accumulate conversation history as context
    
    while True:
        user_input = input("You: ").strip()
        if user_input.lower() in ('quit', 'exit', 'q'):
            print("Goodbye!")
            break
        if not user_input:
            continue
        
        # Format as dialogue — model will try to "complete" the next speaker
        prompt = history + f"\n{user_input}\n"
        prompt_ids = encode(prompt[-800:])  # Keep last 800 chars to stay in context
        
        # Generate response
        full_ids = generate(model, prompt_ids, max_new_tokens=max_response,
                            temperature=temperature, top_k=top_k, device=device)
        full_text = decode(full_ids)
        
        # Extract only the newly generated part
        new_text = full_text[len(decode(prompt_ids)):]
        
        # Take first "paragraph" (until double newline or max 300 chars)
        response_end = new_text.find('\n\n')
        response = new_text[:response_end].strip() if response_end > 0 else new_text[:300].strip()
        
        print(f"Model: {response}\n")
        history = prompt + response

Improvements & Next Steps

Our 10M-parameter model produces recognisable Shakespearean prose but has clear limitations: it can't follow instructions, remember facts, or have real conversations. Here's a roadmap from here to a production-grade assistant:

Path to a Real Chatbot

  1. Better tokenization — Replace character-level with BPE (50k tokens). Each token encodes more meaning, sequences are shorter, training is more efficient. See NLP Part 2: Training Tokenizers from Scratch.
  2. Scale up — Increase d_model=512, n_layers=12, n_heads=16 for ~100M params. Use mixed precision (FP16) and gradient checkpointing to fit in Colab memory. See PyTorch Guide: Mixed Precision Training & Distributed Training.
  3. Better data — Train on a diverse web corpus (OpenWebText, The Pile) rather than Shakespeare alone. Language diversity improves generalisation dramatically.
  4. Instruction fine-tuning (SFT) — Fine-tune on (instruction, response) pairs in Alpaca format. This transforms the language model into an instruction follower.
  5. RLHF — Train a reward model on human preference data, then use PPO or DPO to align responses. This is what makes ChatGPT genuinely helpful. See NLP Part 16: RLHF & Instruction Tuning.
  6. Multilingual extension — Use a multilingual corpus and multilingual SentencePiece tokenizer for a model that handles 100+ languages. See NLP Part 13: Building Multilingual Language Models.
