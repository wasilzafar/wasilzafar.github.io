Physical vs Logical Data Structures

Data structures can be classified based on how they are stored in memory (physical) and how they behave logically.

Physical Data Structures

Physical structures define how data is actually stored in memory:

  • Arrays - Contiguous memory allocation, fixed size at creation
  • Linked Lists - Non-contiguous nodes connected via pointers, dynamic size

Logical Data Structures

Logical structures define the behavior and operations, implemented using physical structures:

  • Linear: Stack (LIFO), Queue (FIFO), Deque
  • Non-Linear: Trees, Graphs
  • Tabular: Hash Tables
# Physical vs Logical Data Structures in Python
import array
from collections import deque

# PHYSICAL: Array - contiguous memory
arr = array.array('i', [1, 2, 3, 4, 5])
print(f"Array: {list(arr)}")

# PHYSICAL: Linked List concept
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

head = Node(10)
head.next = Node(20)
head.next.next = Node(30)

# LOGICAL: Stack (LIFO) - can use list or linked list underneath
stack = []
stack.append(10)  # push
stack.append(20)  # push
print(f"Stack pop: {stack.pop()}")  # Output: 20

# LOGICAL: Queue (FIFO) - using deque for O(1) operations
queue = deque()
queue.append(10)   # enqueue
queue.append(20)   # enqueue
print(f"Queue dequeue: {queue.popleft()}")  # Output: 10

Abstract Data Types (ADT)

An ADT defines what operations can be performed, not how they're implemented. It's the interface, not the implementation.

List ADT Operations

  • insert(index, element) - Insert element at position
  • delete(index) - Remove element at position
  • get(index) - Retrieve element at position
  • set(index, element) - Update element at position
  • length() - Return number of elements
  • search(element) - Find element position

Implementations: Array-based, Linked List-based, Dynamic Array

// ADT Implementation in Java
public interface ListADT<T> {
    void insert(int index, T element);
    T delete(int index);
    T get(int index);
    void set(int index, T element);
    int length();
    int search(T element);
}

// Array-based implementation
public class ArrayList<T> implements ListADT<T> {
    private Object[] data;
    private int size;
    private int capacity;
    
    public ArrayList(int capacity) {
        this.capacity = capacity;
        this.data = new Object[capacity];
        this.size = 0;
    }
    
    @Override
    public void insert(int index, T element) {
        if (size >= capacity) throw new RuntimeException("Array is full");
        for (int i = size; i > index; i--) {
            data[i] = data[i-1];
        }
        data[index] = element;
        size++;
    }
    
    @Override
    public int length() { return size; }
}

Stack vs Heap Memory

Understanding memory is crucial for writing efficient code and debugging memory-related issues.

Aspect Stack Memory Heap Memory
AllocationAutomatic (compile-time)Dynamic (run-time)
SpeedVery Fast (LIFO pointer)Slower (search for free space)
SizeLimited (typically 1-8 MB)Large (available RAM)
ManagementAutomatic cleanupManual / Garbage Collector
StoresLocal variables, function callsObjects, dynamic arrays
ErrorStack OverflowOut of Memory / Memory Leak
 
// C++ Memory Demonstration
#include <iostream>
using namespace std;

void stackExample() {
    // STACK allocation - automatic cleanup
    int localVar = 10;           // Stack: 4 bytes
    int arr[5] = {1,2,3,4,5};    // Stack: 20 bytes (fixed size)
    
    cout << "Stack variable: " << localVar << endl;
    cout << "Stack array[0]: " << arr[0] << endl;
} // localVar and arr automatically freed when function returns

void heapExample() {
    // HEAP allocation - manual cleanup required
    int* ptr = new int(20);           // Heap: single integer
    int* arr = new int[5]{1,2,3,4,5}; // Heap: dynamic array
    
    cout << "Heap variable: " << *ptr << endl;
    cout << "Heap array[0]: " << arr[0] << endl;
    
    delete ptr;       // Free single object
    delete[] arr;     // Free array
}

int main() {
    stackExample();
    heapExample();
    return 0;
}

Static vs Dynamic Memory Allocation

Static Allocation

  • Size determined at compile time
  • Memory allocated on stack
  • Fast but inflexible
  • Cannot resize after creation
int arr[100]; // Fixed 100 elements

Dynamic Allocation

  • Size determined at runtime
  • Memory allocated on heap
  • Flexible but requires management
  • Can resize (with reallocation)
int* arr = new int[n]; // Size n at runtime

Asymptotic Notations (O, Ω, Θ)

Asymptotic notation describes algorithm efficiency as input size approaches infinity, ignoring constants and lower-order terms.

Why Asymptotic Notations?

We use asymptotic notation because actual runtime depends on hardware, language, and implementation details. By focusing on growth rate, we can compare algorithms independent of these factors.

Big-O Notation (O) - Upper Bound

Describes the worst-case scenario. "The algorithm takes at most this much time."

Omega Notation (Ω) - Lower Bound

Describes the best-case scenario. "The algorithm takes at least this much time."

Theta Notation (Θ) - Tight Bound

When upper and lower bounds match. "The algorithm takes exactly this order of time."

# Understanding Asymptotic Notations

def linear_search(arr, target):
    """
    Time Complexity Analysis:
    - Best Case Ω(1): target is at index 0
    - Worst Case O(n): target is last element or not present
    - Average Case Θ(n): target is somewhere in the middle
    """
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

def binary_search(arr, target):
    """
    Time Complexity: Θ(log n) - tight bound for all cases
    Requires sorted array
    """
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

# Test
arr = list(range(1, 101))
print(f"Linear search for 50: index {linear_search(arr, 50)}")
print(f"Binary search for 50: index {binary_search(arr, 50)}")

Space Complexity

Space complexity measures memory usage growth with input size.

# Space Complexity Examples

# O(1) - Constant Space
def sum_array(arr):
    """Uses fixed amount of extra memory regardless of input size"""
    total = 0
    for num in arr:
        total += num
    return total

# O(n) - Linear Space
def duplicate_array(arr):
    """Creates new array proportional to input size"""
    result = []
    for num in arr:
        result.append(num)
    return result

# O(n²) - Quadratic Space
def create_adjacency_matrix(n):
    """Creates n×n matrix"""
    matrix = [[0] * n for _ in range(n)]
    return matrix

print(f"Sum: {sum_array([1,2,3,4,5])}")
print(f"Duplicate: {duplicate_array([1,2,3])}")

Analyzing Complexity from Code

Learn to look at code and determine its time/space complexity.

# Complexity Analysis Patterns

# Pattern 1: Single Loop → O(n)
def single_loop(n):
    count = 0
    for i in range(n):
        count += 1
    return count

# Pattern 2: Nested Loops → O(n²)
def nested_loops(n):
    count = 0
    for i in range(n):
        for j in range(n):
            count += 1
    return count

# Pattern 3: Loop with Halving → O(log n)
def halving_loop(n):
    count = 0
    while n > 1:
        n = n // 2
        count += 1
    return count

# Pattern 4: Nested Loop with Halving Inner → O(n log n)
def n_log_n_pattern(n):
    count = 0
    for i in range(n):
        j = n
        while j > 1:
            j = j // 2
            count += 1
    return count

print(f"Single loop (1000): {single_loop(1000)}")
print(f"Nested loops (10): {nested_loops(10)}")
print(f"Halving loop (1000): {halving_loop(1000)}")

Complexity Rules Summary

  • Drop constants: O(2n) → O(n)
  • Drop lower-order terms: O(n² + n) → O(n²)
  • Sequential operations: O(A) + O(B) = O(A + B)
  • Nested operations: O(A) × O(B) = O(A × B)
  • Different inputs, different variables: O(n × m), not O(n²)

Easy 1920. Build Array from Permutation

Given a zero-based permutation nums, build array ans where ans[i] = nums[nums[i]].

# LeetCode 1920 - Build Array from Permutation
# Time: O(n), Space: O(n)

def buildArray(nums):
    return [nums[nums[i]] for i in range(len(nums))]

print(buildArray([0,2,1,5,3,4]))  # Output: [0,1,2,4,5,3]
print(buildArray([5,0,1,2,3,4]))  # Output: [4,5,0,1,2,3]

Easy 2011. Final Value After Operations

Given array of operations, perform ++X, X++, --X, X-- on initial X=0.

# LeetCode 2011 - Final Value After Operations
# Time: O(n), Space: O(1)

def finalValueAfterOperations(operations):
    x = 0
    for op in operations:
        if '+' in op:
            x += 1
        else:
            x -= 1
    return x

print(finalValueAfterOperations(["--X","X++","X++"]))  # Output: 1
print(finalValueAfterOperations(["++X","++X","X++"]))  # Output: 3
